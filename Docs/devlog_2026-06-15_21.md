# 좀비 헌터 개발로그 (2026.06.15 ~ 2026.06.21)

이번 주는 **"전투의 뼈대"**를 세운 한 주였다.
직업 시스템 → 무기/발사체 → 동료 AI → 동료 섭외 발판까지, 플레이어 한 명이 싸우던 게임을
**"직업을 고르고, 동료를 모아 함께 싸우는"** 구조로 키웠다.

이번 주 핵심 설계 원칙은 하나였다.

> **이동/조준은 "몸"이 하고, 공격 방식은 "직업"이 한다. 그리고 그 몸은 플레이어든 동료 AI든 똑같다.**

이 원칙 덕분에 플레이어용으로 짠 전투 코드를 동료 AI가 거의 그대로 재활용할 수 있었다.

---

## 1. 조작키 — 로스트아크식 마우스 + 모바일 조이스틱

조작을 WASD가 아니라 **로스트아크처럼 우클릭 이동 / 좌클릭 공격**으로 구현했다.

- **우클릭**을 누르고 있는 동안 → 커서가 가리키는 지면으로 이동
- **좌클릭**을 누르고 있는 동안 → 커서 방향으로 자동 공격(`AttackInterval` 간격)

커서가 가리키는 위치는 카메라에서 쏜 광선과 *플레이어 높이의 수평면*이 만나는 점으로 계산한다(`GetCursorGroundLocation`). 커서가 캐릭터에 너무 가까우면 방향이 매 프레임 뒤집혀 제자리에서 진동하는 문제가 있어서, `CursorStopRadius`(기본 60cm) 안에서는 이동을 멈추게 했다.

나중에 모바일에서도 **브롤스타즈처럼 왼쪽=이동 / 오른쪽=공격** 조이스틱으로 조작할 예정이라,
입력을 게임패드 / 터치로 나눠 저장한 뒤 `Tick`에서 합성하도록 미리 분리해뒀다. 터치 조이스틱은 위젯 BP 없이 C++(`UVirtualJoystick`)가 화면에 직접 띄운다.

### 하체/상체 분리 — 전방 워크 애니 1개로 옆걸음 표현
탑다운이라 "다리는 이동 방향, 상체는 조준 방향"을 따로 보여줘야 했다.
액터(몸)는 조준 방향을 보게 두고, AnimBP에서 pelvis를 `+LegYawOffset`, spine_01을 `-LegYawOffset`만큼 돌려 **다리만 이동 방향으로 틀었다.** 덕분에 전방 워크 애니메이션 1개만으로도 옆/뒤로 걸을 때 다리가 이동 방향을 향한다(`LegYawMaxAngle`로 허리가 꺾여 보이는 각도는 제한).

## 2. 직업 시스템 — 검사 / 궁수 / 마법사 / 힐러

직업을 **검사, 궁수, 마법사, 힐러** 4개로 나눴다(와우식). 시작 시 1개를 골라 고정한다.

핵심은 **직업을 "캐릭터"가 아니라 "컴포넌트"로 만든 것**이다(`UJobComponent`).
이동/조준은 소유 캐릭터가 하고, **공격 방식만** 컴포넌트가 책임진다. 소유자가 `ACharacter`이기만 하면 되므로 **플레이어와 동료 AI가 똑같은 직업을 공유**한다.

```cpp
// 직업 베이스 — 공격 방식만 분리. 소유자는 플레이어든 동료든 상관없다.
virtual void Attack();                       // 자동공격 타이밍에 호출 (기본: 몽타주 재생)
virtual void OnAttackNotify(FName NotifyName); // 몽타주 Notify 시점의 실제 타격/발사 판정
```

각 직업은 `Attack()` / `OnAttackNotify()`만 override 한다.

| 직업 | 방식 | 특징 |
|------|------|------|
| **검사**(`USwordsmanJob`) | 근접 | 전방 **구체 스윕**으로 범위 내 적 동시 타격 + 넉백 |
| **궁수**(`UArcherJob`) | 원거리 | 전방으로 **화살(`AProjectile`)** 발사, 빠른 속도 |
| **마법사**(`UMageJob`) | 원거리/범위 | 느린 발사체 + 적중 지점 **범위 폭발(AOE)** — 한 발로 여러 적 |
| **힐러**(`UHealerJob`) | 지원 | "공격"이 곧 **힐 시전** — 전방 아군(자신/동료) 회복, 적엔 무효 |

원거리 직업의 발사 로직은 베이스의 `SpawnProjectileForward()`로 공용화해서 궁수/마법사가 같이 쓴다.

### 무기 메시 교체
직업에 맞춰 손에 든 무기 메시를 바꾼다. 캐릭터(BP)에 이미 붙어 있는 무기 `ChildActorComponent`를 태그(`WeaponComponentTag`)로 찾아 캐시해두고, 직업이 자기 `WeaponMesh`로 교체한다(`SetWeaponMesh`). 지팡이 없는 마법사처럼 메시를 비우면 무기를 숨긴다.

## 3. 전투 공통 베이스 — ACombatCharacter

플레이어/동료/적은 공통분모가 많았다 → **HP·데미지·죽음/부활 전환·공격 몽타주 Notify 배선**을 베이스 `ACombatCharacter`로 모았다.

```
ACombatCharacter (Abstract)
 ├── AMyPlayer    (입력으로 구동)
 ├── ACompanion   (AI로 구동)
 └── AEnemy
```

서브클래스는 차이 나는 부분만 구현한다.
- `HandleAttackNotify()` — 몽타주 Notify가 들어왔을 때 실제 타격을 어떻게 줄지(플레이어/동료는 현재 직업의 `OnAttackNotify`로 넘김)
- `OnDeath()` / `OnRevive()` — 죽거나 풀에서 되살아날 때 무엇을 끄고/켤지

HP가 0 이하가 되면 `SetHP`가 자동으로 `IsDead`를 켜고 죽음/부활 훅을 전환 시점에 딱 1번씩 호출한다.

## 4. 동료 AI — 따라다니다 적이 보이면 싸운다

동료(`ACompanion`)는 평소엔 리더(플레이어)를 졸졸 따라다니고(Following), 탐지 반경(`DetectRadius`) 안에 적이 들어오면 다가가 공격한다(Fighting).

전투/조준은 **플레이어와 똑같은 `UJobComponent`를 재활용**한다 — 입력 대신 AI가 구동할 뿐이다. 직업 시스템을 컴포넌트로 뺀 설계가 여기서 그대로 보상으로 돌아왔다. 근접/원거리 교전 거리는 직업별 `EngageRange`로 구분한다.

작업하면서 잡은 버그들:
- 죽은 적을 계속 공격하던 버그 (`FindNearestEnemy`가 "살아있는" 적만 찾도록)
- 적 오브젝트 풀에서 이미 파괴된 액터를 참조해 생기던 크래시 (풀 가드 추가)

## 5. 오픈월드 동적 NavMesh

맵이 무한 청크로 생성되다 보니 NavMesh를 통째로 깔 수 없었다.
플레이어에 `UNavigationInvokerComponent`를 붙여 **플레이어 주변에만 NavMesh를 동적으로 생성**하게 했다. 동료/적 AI는 이 위에서 길찾기를 한다.

## 6. 동료 섭외 이벤트 — 밟으면 차오르는 발판

쇼츠형 게임처럼 **발판을 밟으면 게이지가 차고, 가득 차면 동료가 소환**되는 이벤트를 구현했다(`ACompanionSpawnZone`).

![](https://velog.velcdn.com/images/gestgest/post/fd2e50d7-5874-4158-818b-0bb25f508bdd/image.png)

처음엔 단순히 "올라서면 시간에 비례해 게이지가 찬다"였는데, 작업하면서 **"돈을 내야 게이지가 찬다"**로 바꿨다.

- 발판 위에 서 있으면 `PaymentInterval`마다 `MoneyPerPayment`씩 돈을 소비한다(`TrySpendMoney`).
- 누적 결제액(`PaidMoney`)이 `MaxMoney`에 도달하면 게이지 완성 → 플레이어의 `RecruitCompanion()` 호출(C키 섭외와 **동일 로직 재사용**).
- 돈이 부족하면 게이지가 안 오르고 `OnInsufficientFunds`로 "돈 부족" 연출.
- 발판에서 내려와도 **이미 낸 돈만큼의 게이지는 유지**된다(낸 돈이 아까우니까).
- `bOneShot`(일회성) / `Cooldown`(재충전 대기)로 작동 방식을 조절.

UI는 `OnProgressChanged` / `OnZoneCompleted`(BlueprintImplementableEvent)로 위젯에 연동하고, BP 위젯이 없어도 `bShowDebugGauge`로 발판 위에 디버그 게이지 바를 바로 띄울 수 있게 했다.

> 핵심은 C키 디버그 섭외와 발판 섭외가 **같은 `RecruitCompanion()`을 부른다**는 점. 소환 방식(발판/키)이 늘어나도 동료 생성·배치 로직은 한 곳만 고치면 된다.
